Тема 3.2. Практическое занятие

 

ОБУЧАЮЩЕЕ ВИДЕО

 

Цель: научиться создавать простую базу данных, моделирующую работу врача с пациентами: приём, диагнозы, рецепты и анализы.

Задание в Access:

1.    Создать базу данных в Access с названием АРМ_врача.accdb.

2.    Создать три таблицы:

Таблица 1: Пациенты

Поле

Тип данных

ID_пациента

Счетчик (ключ)

ФИО

Краткий текст

Дата_рождения

Дата/время

Пол

Краткий текст

Адрес

Краткий текст

Телефон

Краткий текст

Таблица 2: Приёмы

Поле

Тип данных

ID_приёма

Счетчик (ключ)

Дата

Дата/время

ID_пациента

Числовой

Диагноз

Краткий текст

Назначение

Длинный текст

Поле ID_пациента — внешний ключ, связывается с таблицей Пациенты.

Таблица 3: Анализы

Поле

Тип данных

Результат

Длинный текст

Установи связи между таблицами:

  • Таблица Пациенты — главная.
  • Таблицы Приёмы и Анализы — подчинённые.
  • Связь по полю ID_пациента.

Ввести данные:

Ввести минимум:

  • 3 пациента,
  • по 2 приёма для каждого,
  • по 1 анализу на каждого пациента.

Создать запросы:

1.    Пациенты с диагнозом "ОРВИ"

2.    Анализы, сделанные после определённой даты

3.    Все приёмы одного конкретного пациента (по фамилии)

Создать форму для ввода новых пациентов.

Создать отчёт: "Список приёмов на сегодня"

Дополнительное (по желанию):

  • Добавь поле "Электронная почта" в таблицу Пациенты.
  • Настрой подстановку пола через список.
  • Добавь кнопку "Новый приём" на форме.

Вопросы для размышления после выполнения:

1.    Почему важно хранить данные в базе, а не на бумаге?

2.    Что будет, если потерять бумажные карты?

3. Как связи между таблицами помогают врачу быстрее находить нужную информацию?
